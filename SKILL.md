# 访谈式简历生成器

通过智能访谈引导用户生成专业简历，使用 AskUserQuestion 工具进行交互式问答，最终输出 Markdown 格式简历文件。

## 触发场景

- 用户说"帮我写简历"、"生成简历"、"做一份简历"
- 用户说"我要找工作，需要简历"
- 用户说"更新一下我的简历"
- 用户说"准备面试，需要简历"
- 使用 `/interview-resume-generator` 命令

## 核心流程

```
阶段0: 场景识别 → 阶段0.5: 行业识别 → 阶段1: 基本信息 → 阶段2: 教育背景 → 阶段3: 工作经历 → 阶段4: 项目经历 → 阶段5: 技能评估 → 阶段6: 自我评价 → 生成Markdown简历 → 阶段7: 生成HTML版本（可选）
```

## 阶段0: 场景识别

**目标**：确定求职场景，决定后续访谈策略

**问题设计**：

```
问题: "你好！我是简历小助手，接下来我会通过几个问题帮你生成一份专业简历。首先，请问你目前处于什么求职阶段？"

选项:
1. label: "社招 - 有工作经验，跳槽求职"
   description: "重点展示工作成果、项目经验、技能深度"

2. label: "校招 - 应届生/实习生"
   description: "重点展示教育背景、实习经历、学习能力"

3. label: "转行 - 跨行业发展"
   description: "重点展示可迁移技能、学习意愿、转型规划"
```

**策略记录**：
- `scenario_type`: social_recruit / campus_recruit / career_change
- 根据选择加载对应简历模板和问题侧重

## 行业定制化追问策略

根据 `industry_type` 在工作经历和项目经历阶段使用不同的追问策略：

### 技术研发类（tech）

**核心关注点**：技术深度、架构能力、性能优化、技术难点攻克

**工作经历追问**：
```
问题: "在这个岗位上，你遇到的最大技术挑战是什么？如何解决的？"
```

```
问题: "请说说你使用的主要技术栈，以及你在技术选型上的考量"
```

**项目经历追问**：
```
问题: "这个项目的技术架构是怎样的？你为什么选择这个架构？"
```

```
问题: "项目中有哪些性能优化或技术亮点？\n\n提示：\n- 性能提升（响应时间、吞吐量）\n- 稳定性改进（可用性、容错）\n- 技术创新（新方案、开源贡献）"
```

**成果量化提示**：
- 性能提升：响应时间、QPS、并发量
- 稳定性：可用性99.9%、故障率降低
- 代码质量：覆盖率、Bug率
- 工程效率：构建时间、部署频率

---

### 产品类（product）

**核心关注点**：用户洞察、数据驱动、产品规划、跨部门协作

**工作经历追问**：
```
问题: "你负责的产品/功能，核心用户需求是什么？你是如何发现和验证的？"
```

```
问题: "说说你做过的最重要的产品决策，依据是什么？"
```

**项目经历追问**：
```
问题: "这个产品/功能上线后，数据表现如何？你是如何定义成功的？"
```

```
问题: "在产品迭代过程中，有没有遇到用户反馈与预期不符的情况？你是如何调整的？"
```

**成果量化提示**：
- 用户指标：DAU/MAU、留存率、转化率
- 业务指标：GMV、营收、ARPU
- 产品指标：功能使用率、NPS、满意度
- 迭代效率：需求交付周期、迭代频率

---

```

    ├── resume_modern.html          # 现代简约风
    ├── resume_classic.html         # 经典商务风
```

### 7.3 HTML模板说明

**8种模板风格：**

| 模板 | 文件名 | 特点 |
|------|--------|------|
| 🎨 现代简约 | resume_modern.html | 蓝紫渐变，清新专业 |
| 💼 经典商务 | resume_classic.html | 深色稳重，金边装饰 |
| ✨ 创意活力 | resume_creative.html | 紫粉渐变，圆角设计 |
| 📄 极简纯净 | resume_minimal.html | 黑白灰调，简约至上 |
| 🌷 优雅格调 | resume_elegant.html | 粉金配色，优雅温柔 |
| 💻 科技极客 | resume_tech.html | 暗黑主题，代码风格 |
| 📋 正式传统 | resume_formal.html | 深蓝稳重，正式专业 |
| 🌈 活力多彩 | resume_colorful.html | 彩虹渐变，年轻动感 |

### 7.4 HTML功能特性

每个HTML模板都包含：

1. **模板切换**：点击"切换模板"按钮，在8种风格间自由切换
2. **导出PDF**：点击"导出PDF"按钮，调用浏览器打印功能
3. **导出PNG**：点击"导出PNG"按钮，使用html2canvas生成高清图片
4. **响应式设计**：适配手机、平板、电脑等不同设备
5. **打印优化**：打印时自动隐藏工具栏，优化输出效果

### 7.5 HTML生成提示词

生成HTML时，使用以下结构：

**主HTML文件（{姓名}_简历.html）**：
- 使用现代简约风格（modern）作为默认风格

## 回答质量评估机制

### 评分维度

| 维度 | 分值 | 判断标准 |
|-----|------|---------|
| 具体性 | 0-3分 | 是否有具体的场景、角色、任务描述 |
| 量化程度 | 0-3分 | 是否有数字、比例、具体成果 |
| 完整性 | 0-2分 | 是否回答了问题的核心要点 |
| 亮点度 | 0-2分 | 是否有独特价值、突出成就 |

### 响应策略

| 总分 | 等级 | 行为 |
|-----|------|-----|
| 0-3 | 低 | 必须追问，提供示例引导 |
| 4-6 | 中 | 适度追问，挖掘更多细节 |
| 7-8 | 良 | 简单确认，记录亮点 |
| 9-10 | 优 | 深度挖掘，成为简历核心卖点 |

### 质量判断示例

**低质量回答**：
- "我负责开发" → 太笼统，必须追问
- "做了很多项目" → 没有具体内容，必须追问
- "工作很努力" → 没有实际成果，必须追问

**高质量回答**：
- "负责抖音电商首页，使用React+TS重构，将首屏加载时间从3s优化到1.2s" → 具体且有量化
- "带领8人团队完成支付系统重构，日处理订单从10万提升到100万" → 有角色、有成果

---

## 注意事项

1. **渐进式提问**：一次只问一个问题，不要一次问太多
2. **保持友好**：使用"请"、"可以"等礼貌用语
3. **提供示例**：每个问题都提供填写示例
4. **允许跳过**：用户可以跳过不合适的问题
5. **智能追问**：根据回答质量决定是否追问
6. **数据保护**：不存储敏感信息，生成后即删除
